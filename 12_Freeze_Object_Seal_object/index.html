Object.seal() Vs Object.freeze()

Object.seal() provides a way to lock down an objectâ€™s properties while still allowing modifications to their values.

  1. Sealing an Object:
    • It prevents new properties from being added to the object.
    • Existing properties become non-configurable (cannot be removed or have their enumerability and configurability changed).
  2. Behavior:
    • Values of existing properties can still be changed if they are writable.
    • The method returns the same object that was passed in.

Object.freeze() locks down an object, making it read-only and preventing any modifications.
